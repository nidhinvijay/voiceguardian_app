# VoiceGuardian App

The Flutter app is the user-facing client of VoiceGuardian. It uses Provider state management, Firebase Cloud Messaging, and Agora RTC to provide authentication, friend discovery, real-time voice calls, transcription-driven coaching, and historic insights.

Repository URL: https://github.com/nidhinvijay/voiceguardian_app.git

## Prerequisites
- Flutter SDK 3.22+ (bundled with Dart 3.9) – confirm with `flutter doctor`
- Android Studio or VS Code with Flutter/Dart plugins
- Xcode 15+ if you plan to target iOS
- A Firebase project with Cloud Messaging enabled (push notifications drive incoming calls)
- Agora project with App ID and App Certificate configured in the backend `.env`
- A running VoiceGuardian backend (local server, ngrok tunnel, or deployed instance)

## Quick Start
```bash
git clone https://github.com/nidhinvijay/voiceguardian_app.git
cd voiceguardian_app
flutter pub get
flutter run
```

## Configure the API Base URL
The app reads its backend endpoint from `lib/utils/constants.dart`.

```dart
class Constants {
  // Use 10.0.2.2 for Android Emulator to connect to localhost
  // static const String baseUrl = "http://10.0.2.2:8000/api/v1";
  static const String baseUrl = "https://criselda-endurable-nonsociably.ngrok-free.dev/api/v1";
  static const String agoraAppId = "91496d742dca43e596046340b9e4b4ad";
}
```

- **Hosted backend**: keep the deployed URL or replace with your production host.
- **Local FastAPI + emulator**: uncomment the `10.0.2.2` line.
- **Local backend + device**: set `baseUrl` to your computer’s LAN IP and ensure the device can reach it.
- After changing the base URL, hot restart the app so the HTTP client picks up the new value.

## Firebase Setup
- `lib/firebase_options.dart` is generated by `flutterfire configure`. Regenerate it if you switch Firebase projects.
- Android: keep `android/app/google-services.json` matched to your Firebase project.
- iOS: add `GoogleService-Info.plist` inside `ios/Runner/` and register it in Xcode (Runner target → Build Settings).
- Additional platforms (web, desktop): re-run `flutterfire configure`.

## Running on Android
- Launch an emulator or connect a device and run `flutter run`.
- The app requests microphone permission for Agora voice calls and POST notifications for FCM.
- The Flutter UI handles incoming calls by listening for `incoming_call` pushes and showing a custom screen.

## Running on iOS
- Open `ios/Runner.xcworkspace` in Xcode, configure signing, and run on a simulator or device.
- Enable Push Notifications and Background Modes → Remote notifications under the Runner target.
- Ensure `GoogleService-Info.plist` is present before building.

## Directory Tour (`lib/`)
- `lib/main.dart` – boots Firebase, registers FCM listeners, and wires the global providers that connect auth, call state, Agora, and transcription services.
- `lib/providers/auth_provider.dart` – handles registration/login, persists the JWT, and registers the device’s FCM token with the backend.
- `lib/services/api_service.dart` – one place for every HTTP request (auth, friends, call lifecycle, transcription/tts).
- `lib/services/agora_call_service.dart` – initializes Agora, joins/leaves channels, toggles mute/speaker, and feeds raw microphone audio into the transcription websocket.
- `lib/services/transcription_service.dart` – maintains the `/calls/transcribe_audio` WebSocket, decodes rephrases, and exposes callbacks for UI to play coaching audio.
- `lib/services/call_state_service.dart` – tracks the lifecycle (idle, ringing, connected, ended) so the app can coordinate FCM signals with UI state.
- `lib/screens/auth_wrapper.dart` – shows either the main navigation shell or login flow depending on authentication state.
- `lib/screens/main_shell.dart` – bottom navigation housing the home, history, friends, and profile tabs.
- `lib/screens/home_screen.dart` – lists friends, allows outgoing calls, and kicks off the Agora + transcription flow.
- `lib/screens/incoming_call_screen.dart` – full-screen incoming call UI (triggered by FCM) with accept/decline actions wired to the backend.
- `lib/screens/call_screen.dart` – active call UI, Agora controls, and TTS-based coaching audio embedded via transcription results.
- `lib/screens/history_screen.dart` – displays call history returned by `GET /calls/history`.
- `lib/screens/friends_screen.dart` – send friend requests and accept pending invites.
- `lib/screens/profile_screen.dart` – shows username, phone, and respectfulness score.

## Troubleshooting
- Authentication issues: verify `Constants.baseUrl`, ensure the backend is reachable (`curl` or `httpie` the login endpoint), and check the response body for helpful `detail` messages.
- Incoming calls never appear: make sure the device registered its FCM token (`lib/providers/auth_provider.dart`) and the backend sends the `incoming_call` payload.
- Audio hiccups: confirm microphone permissions are granted, the Agora App ID matches the backend, and network latency is acceptable.
- Server errors: check the FastAPI logs at `/runs` (or the terminal running `uvicorn`) for `HTTPException` details.
